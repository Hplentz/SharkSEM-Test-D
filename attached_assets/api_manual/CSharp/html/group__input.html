Detailed Description

External SharkSEM application can obtain user input from the Tescan Essence software. Arbitrary label and input overlay window is displayed in the Tescan Essence software.

There are no priorities among the remote sessions, there is only single input dialog. "The last wins" strategy takes place.

Every input dialog can be managed with three functions. This is the recommended usage:

  1. Configure and open a new dialog for user input using a function with Dlg suffix.
  2. Check the status of the dialog via InputDlgStatus() function, until the dialog is either accepted or rejected.
  3. If the dialog was accepted, a valid value entered by the user can be acquired using a function with Get suffix.

Input dialog has been introduced in version 3.1.11

Acquires status of last opened Input dialog.

The Status value currently expresses two flags - dialog accepted and dialog open. Their bitwise positions and value meanings are described in the table below.

Bit position 1 0
Flag Dialog accepted flag Dialog open flag
Bit value 1 0 1 0
Flag value Accepted Rejected Dialog open Dialog closed

Valid flag combinations result in following Status values.

Status value Binary value Flags Status
0 0b0000 Rejected, Closed No dialog has been open yet / last dialog was rejected
1 0b0001 Rejected, Open Waiting for user input
2 0b0010 Accepted, Closed Last dialog was accepted, a valid value can be acquired
Return values
Statusflags of the last input window
Timing
Executed immediately.
