Detailed Description

Remote control of chamber cameras. Most SEMs are equipped with chamber camera - chamberscope. Remote client can ask for video from the camera. The video consists of static images sent in regular intervals.

Open camera window in the SEM software, enable the chamberscope image stream.

The camera image size is not fixed, the actual size is sent in the image callback. Client can ask for image with reduced size (if zoom < 1.0).

Camera pictures are sent through the data channel (same as the scanning data). You can retrieve them by calling FetchCameraImage().

Pictures can be retrieved only from single camera at time. If CameraEnable() is called during previous camera stream is running, it stops previous stream and starts a new one.

The channel parameter is used since protocol version 3, in previous versions must be 0. Channel indexes can be retrieved by CameraEnum().

Parameters
Indexcamera index, used since version 3
Zoomzoom (0.05 - 1.0)
Fpsframes per second (0.1 - 5.0)
Compressionreserved, must be 0
Timing
If the Chamber View module in the SEM software is closed, it may take several seconds till the client receives valid image. If the Chamber View module is running, the first image is sent immediately.
Call Context
Call CameraEnable() only if camera is disabled.

The returned map (string) has the following form:

camera.0.name=
camera.1.name=
...
camera.N-1.name=

Where N is camera count. For future compatibility, application must silently ignore any key which does not have exactly the above form.

The camera number can be used in CameraEnable() as channel index.

CameraData()
Image is uncompressed, grayscale, sent in a byte array. The data starts with top-left corner. The camera image is re-sampled according the client's request. Each image is sent at once, in a single message.
The CameraData() messages are sent through the data channel (same as the scanning data). You can retrieve them by calling FetchCameraImage().
void CameraData(
in int channel,
in int bpp,
in int width,
in int height,
in char[] data
);
Parameters:
  • channel: channel index (reserved, should be 0)
  • bpp: bits per pixel (always 8)
  • width: image width
  • height: image height
  • data: image data, width * height bytes
