Detailed Description

This chapter describes interface of EDX scanning. There are several scanning methods available - XY Scan, Line Scan and Map Scan.

Scan series of horizontal lines EDX mode. EDX mode allows synchronizing with EDX DPP or specifying dwell time depending on image brightness.

PointListlist of points delimiting the horizontal lines
SendData1 - send image data to client, 0 - discard image data
Single0 - continual scanning, 1 - single frame
Returns
0 - ok, < 0 - failed, invalid parameters
Timing
Executed immediately. Wait A flag is set during scanning. The flag is cleared when single scan has finished or when ScStopScan() is called.

ScData() is called periodically when image data is ready, the period is about 50 ms.

Remarks
Refer to ScScanXY() and ScScanEDXXY() functions. This function is very similar to ScScanEDXXY(), but instead of regular rectangular region, arbitrary region(s) consistion of horizontal lines are scanned.

See ScScanEDXXY() for parameter description.

The PointList is a list of 32-bit unsigned values. Each pair (0-1, 2-3, ...) defines first and last pixel of a scan line. Both limits are included into the line. The pixels are addressed sequentially from top left corner. The scan line must not cross the image row boundary. There are no restrictions how lines are organized in the list.

The maximum number of items (lines) in the PointList is 100 000. This is guaranteed since protocol version 2.0.18, previous versions can accept less items.

The SyncMode parameter has more options than other EDX scanning functions. The bits 0 - 7 have the same meaning as usual, i.e. define brightness-based or EDX-based triggering. The bits 8 - 15 specify behavior of the beam flybacks:

  • 0 - direct flybacks, faster, higher hysteresis artifacts
  • 1 - flybacks as in XY mode, slower, lower hysteresis artifacts The bits 8 - 15 have been introduced in SharkSEM version 2.0.17.
Call Context
Scanning must be inactive when calling ScScanEDXMap(). No automatic procedure should be running. Make sure the wait flags are set properly.

Timing
Executed immediately. Wait A flag is set during scanning. The flag is cleared when single scan has finished or when ScStopScan() is called.

ScData() is called periodically when image data is ready, the period is about 50 ms.

Remarks
Refer to ScScanXY() function. This is its variant used for EDX mapping.

The area is defined the same way as in ScScanXY(). The image data are sent to client in the same way as well. What's different is how pixel dwell time is controlled.

Mode 0 - dwell time depends on the brightness

Each pixel's brightness is evaluated first. If the pixel brightness is in the range <ThrLow; ThrHigh>, dwell time is DwellBright. If it is outside the range, the dwell time is DwellDark instead. Since version 3.1.6, ThrLow can be greater than ThrHigh. In such case, the condition is valid if brightness is greater than ThrLow or less than ThrHigh.

Mode 1 - dwell time depends on external synchronization

Typically, this is used with EDX DPP. Each pixel's brightness is evaluated first. If the pixel brightness is in the range <ThrLow; ThrHigh>, the scanning proceeds to the next pixel only after external synchronization pulse is received (eg. after processing 10 000 X-Ray counts). Since version 3.1.6, ThrLow can be greater than ThrHigh. In such case, external synchronization takes place if if brightness is greater than ThrLow or less than ThrHigh.

Both mode 0 and mode 1 require that pixel brightness is evaluated first. In each pixel, WaitCount * WaitDwell acquisitions are done. Brightness of the last acquisition is used for evaluation of the condition. Note that signal propagation after beam deflection takes arbitrary time, typically few microseconds.

ScData() is called when image data is sent to the client. If 16-bit scanning is active, one pixel consists of two bytes, the bytes are stored in little-endian order.

Call Context
Scanning must be inactive when calling ScScanEDXXY(). No automatic procedure should be running. Make sure the wait flags are set properly.
