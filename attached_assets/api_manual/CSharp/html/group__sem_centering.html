inlinestatic

Start one pass automatic objective alignment procedure.

Channel index is a zero-based index of the input channel. Appropriate detector must be selected as an active.

There are several important preconditions. If they are not fulfilled, the objective centring may fail.

  • Good signal quality
  • Good structure on the specimen surface
  • Focused image

If the dwell parameter is omitted, current scanning speed is used instead. Refer to ScSetSpeed(). It is recommended that all new client applications use the Dwell parameter instead of the ScSetSpeed() call.

Current look up table and LUT source is used, refer to ScLUTParSet and ScLUTSrcSet. If no LUT is used, default values are used: min = 0, max = 100, gamma = 1.

As a result, objective centering value is changed. Unfortunately, it is not possible to automatically verify the quality of the automatic alignment result.

Parameters
Channelinput video channel
Dwelldwell time in [ns], optional
Return values
Result0 - ok, -1 - unknown error, -2 - error: scanning is active, -3 - failed to start, -4 - failed
Timing
It will wait until procedure is done (few seconds). Wait D flag is set during the movement procedure.

static void SharkSEM.Commands.EnumCenterings ( this Connection  Conn,
out string  Map

inlinestatic

Get list of all column centering parameters.

The map has the following form:

cen.0.name
cen.0.count
cen.0.unit
cen.1.name
...
cen.X.name
cen.X.count
cen.X.unit

The key is separated by two dots. The middle number is an index of the centering parameter. Application should not expect that the indexes form a consecutive sequence. The index can be any integer number.

  • name: human-readable name,
  • count: 1 or 2 (one if only the first value is used, two otherwise)
  • unit: %, mm, or other physical unit

The exact set of centering parameters depends on the SEM configuration.

For future compatibility, application must silently ignore any key which does not have exactly the above form.

Return values
Mapstring containing centering parameters
Timing
Executed immediately.

inlinestatic

Get column centering parameter.

This function returns one or two values (the y parameter may not be used in some cases).

The SEM column centering depends on:

  1. accelerating voltage
  2. scanning mode
  3. active detector

Each accelerating voltage range (index) keeps its own centering set. Each scanning mode keeps its centering set as well. Finally, there are some detectors (In-beam) which have an effect on centering as well and must keep an extra set.

Note
It is usually not necessary to read/write the centering from the remote application.

Set column centering parameter.

If the values are out of range, they are limited.

Note
It is usually not necessary to read or adjust the centering from the remote application.
Parameters
Indexindex of the parameter
Xcentering value in X axis
Ycentering value in Y axis
Storestore centering values into interpolation engine
  • 1 - store (default) - centering is remembered in an interpolation engine
  • 0 - don't store - centering is not remembered in an interpolation engine, on the next optics change the centering value is lost
Timing
The execution time may vary. If waiting for C flag is not requested, it is executed immediately. If waiting C flag is requested, it will wait until procedure is done. Wait C flag is set until the system becomes stable (few milliseconds).
