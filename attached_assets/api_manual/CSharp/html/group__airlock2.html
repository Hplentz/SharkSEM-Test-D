Detailed Description

Motorized airlock allows fully automated specimen exchange. The SharkSEM API is different from the manual airlock. Functions described in this chapter can only be used if airlock type 2 is installed.

Warning:
Avoid using WaitFlags when applying Arl2 commands as they may cause improper command functioning.

◆ Arl2Calibrate()

static void SharkSEM.Commands.Arl2Calibrate ( this Connection  Conn)
inlinestatic

Calibrate the airlock manipulator.

The whole procedure takes tens of seconds, use Arl2GetStatus() to find out when the procedure is finished.

This function is not necessary during normal operation. This is actually a variant of Arl2Vent() function, unlike Vent, is also does calibration of the airlock manipulator.

Timing
This is a non-blocking call, it is executed immediately.
Call Context
Airlock must be in the "inside" position.
Version
2.0.5 and later

  • the whole part after division

6 Load Lock arm is moving from the Load Lock chamber to the home position
7 LSV is closing
8 Load Lock chamber is venting
9 Load lock arm is moving from the home position to the outside position + calibration
10 Load Lock is moving to the nearest endswitch (only for recovery, none endswitch is pushed and LSV is closed)

◆ Arl2Pump()

static void SharkSEM.Commands.Arl2Pump ( this Connection  Conn)
inlinestatic

Load the specimen into the airlock chamber and pump the chamber.

In the first step, the specimen is loaded into the airlock chamber. In the second step, the chamber is pumped down. The whole procedure takes tens of seconds. Use Arl2GetStatus() to find out when the pumping is finished.

The call has no effect if the airlock is already evacuated.

Timing
This is a non-blocking call, it is executed immediately.
See also
VacGetStatus()
Arl2GetStatus()
Call Context
The main SEM chamber should be pumped and SEM vacuum status should be "ready" before this call. The airlock must be in the "outside" position before this call.

int  Mode,
int  LoadLockSample,
out int  Result 
)
inlinestatic

Error recovery procedure.

Arl2Recovery() is used in case of airlock manipulator crash or manual stop.

The recovery process depends on the state of the separation valve (LSV).
1) If separation valve is closed:

  • The loadlock arm is outside the microscope chamber and the recovery can be processed automatically.
    • The user can specify the location of the sample after the recovery procedure, either in the loadlock chamber (unloaded) or in the microscope chamber (loaded), using the 'LoadLockSample' parameter.
    • The recovery process ends with calibrated stage


2) The separation valve is opened:

  • The loadlock arm can be located inside the microscope chamber. The user can specify whether the recovery should be processed manually or automatically using the 'Mode' parameter.
  • The user can specify the location of the sample after the recovery procedure, either in the loadlock chamber (unloaded) or in the microscope chamber (loaded), using the 'LoadLockSample' parameter.
  • The recovery process ends with uncalibrated stage

Executing the Arl2Recovery() command without any parameters initiates automatic recovery mode, assuming the sample is located inside the microscope chamber.

Recovery Modes:

1) Manual mode:

  • The microscope chamber will be vented
  • The stage will be uncalibrated

The manual recovery process is completed by venting the microscope chamber. The loadlock position remains unchanged and is located within the microscope chamber. Note that the LSV is open and the stage will be uncalibrated at this time. The manual mode is used when the loadlock and stage have crashed together and the user must resolve the issue manually. If everything appears to be in order, the user can then run the recovery process automatically.

Warning:
If the stage is located on the chamber door and the loadlock gets stuck within the stage or interlocked with the carousel on the stage, opening the door without caution may cause damage to the loadlock arm. If needed, move the stage away from the loadlock arm using the Stage Control panel while observing the Chamber View live camera view in Tescan Essence software.

2) Automatic mode:
a) if LSV is opened:

  • The loadlock arm will be retracted from the microscope chamber to its loadlock chamber Home position
  • The LSV will be closed
  • The loadlock chamber will be vented
  • The stage will be uncalibrated
    b) if LSV is closed:
  • The loadlock arm will be retracted to its loadlock chamber Home position
  • The loadlock chamber will be vented
  • The stage stays calibrated

    The automatic recovery process finishes with pumped microscope chamber, closed LSV and vented loadlock chamber with retracted loadlock arm. Note that if the LSV was opened, the stage will be uncalibrated.

If manual mode is selected and the loadlock can be recovered automatically (closed LSV), the recovery will finish with an error.

Parameters
Mode- Recovery mode
  • 1 - manually
  • 2 - automatically
LoadLockSample- Where is the sample?
  • 1 - unloaded (in loadlock chamber)
  • 2 - loaded (in microscopy chamber)
Return values
Result
  • 0 - ok
  • -1 - manual mode is not enabled
  • -2 - recovery error
  • -3 - stage is not calibrated
  • -4 - no defined position for given substage
  • -5 - any other loadlock has already been running
  • -6 - loadlock has already been running
  • -7 - target position is not defined
  • -8 - decontaminator is running
  • -9 - decontaminator has been started
Timing
This is a non-blocking call, it is executed immediately.
Call Context
In the "inside" position, "outside" position, or in case of "error" status.
Version
2.0.5 and later
3.x.x
