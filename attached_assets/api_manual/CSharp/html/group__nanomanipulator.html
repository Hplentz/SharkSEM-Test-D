Start nanomanipulator movement, specify velocity.

Infinite movement is started. Application must stop it later using NManipStop() call.

If any arguments are missing, the velocities are zero.

This function is supported by all types of nanomanipulators. It is recommended to use watchdog timer together with NManipMove(). Refer to NManipWatchdog().

Warning
Nanomanipulators without position indicators will run infinitely. If the nanomanipulator is forced to move for a long time after it hits one of the mechanical limits, it can be damaged.
Parameters
Indexnanomanipulator index
Xx velocity [mm/s]
Yy velocity [mm/s], optional
Zz velocity [mm/s], optional
Rotrotation velocity [degrees/s], optional
Return values
Result0 - ok, -1 - not available, -2 - not calibrated, -3 - collision detected, -4 - movement failed, -5 - disabled, -6 - already moving
Timing
Executed immediately, but movement takes some time.
See also
NManipWatchdog()

Nanomanipulator must be idle. If the nanomanipulator is busy, the command is ignored.

Get list of available Nanomanipulators.

Map of nanomanipulators is returned. The map has the following form:

nanomanipulator.0.name=name0
nanomanipulator.1.name=name1
...
nanomanipulator.N-1.name=nameN-1

Where:

  • N is number of nanomanipulators.
  • nameX is human-readable nanomanipulator name.
  • X is a numeric nanomanipulator index.

For future compatibility, application must silently ignore any key which does not have exactly the above form.

Return values
Maplist of nanomanipulators
Timing
Executed immediately.
Call Context
Anytime

int  Index,
double  Timeout 
)
inlinestatic

Nanomanipulator safety watchdog timer.

Nanomanipulator watchdog timer is a safety feature. When nanomanipulator movement is started from remote application, it may be hazardous if the remote application crashes or if the connection is lost, because the movement won't stop.

Before starting the movement, application may schedule watchdog timer, which stops the movement when it times out. The application must periodically restart the timer, indicating that it is still alive.

The timer is restarted by calling NManipIsBusy. When NManipIsBusy reports "in motion" status, timer is internally restarted. When it reports "stopped" status, the timer is killed.

Once NManipWatchdog() is initiated, application is responsible for calling NManipIsBusy periodically, in a shorter time than specified by timeout. The timer is killed either if NManipIsBusy reports that the movement has been finished or if NManipStop() is called explicitly.

Example:

NManipWatchdog(2.0); // schedule watchdog timer to 2 s
NManipMoveTo(10, 20); // movement command
while (NManipIsBusy()) { // restart/kill the timer
if (stop_requested) { // if stop requested by user
NManipStop();
break;
}
Sleep(1000); // wait a while
}

Indexnanomanipulator index
Timeouttimeout [s]
Timing
Executed immediately.
Call Context
Just before movement is started.
