Detailed Description

Preset is a user-defined group of parameters (stage position, scanning mode, detector configuration, ...). The presets can only be created by the user. Each preset is identified by unique, user-defined name. SharkSEM can enumerate the presets and activate one of them.

In version 2.x.x presets works as SEM presets and for FIB there are specialized functions. Since version 3.0.0 there are separated SEM, FIB and general preset function sets. Both SEM and FIB presets are sub-sets of general presets, for more information about them please refer to SEM Presets and FIB Presets.

Get list of available presets.

The returned string contains names of all presets (if Group is not set) or presets belonging to a group (if Group is set). The string is divided into lines, there is one preset name per line. If there are no presets, the string is empty.

Both system and user presets are listed. If both contain an identical name, user preset is preferred.

Activate preset.

The preset table is searched for the specified preset name. If it is found, the preset is activated. If the preset is not found, the request is ignored.

If given name exists in both user and system presets, user preset is activated.

Parameters
Idname of the preset to be activated
Timing
This call can take arbitrary time, depending on the previous state of the system. Before further processing, check wait flags using IsBusy() call, or use conditional command execution.
Call Context
Depends on the operating state. For example, it is not possible to activate preset containing stage movement, if stage is already busy.

Update the values of a given preset.

The user presets are searched for the specified preset name. If the preset is found, the preset is updated and saved. Otherwise, the appropriate error is returned.

Note: This command can't be used to update system presets.

Parameters
Idname of the preset to be updated
Return values
Result0 - SUCCESS, 1 - USER_PRESET_NOT_FOUND, 2 - USER_PRESET_UPDATE_FAILED, 3 - USER_PRESET_VALIDATION_FAILED, 4 - FATAL_ERROR
Timing
Executed immediately.
Call Context
In order to get reproducible results, the state of the microscope should be stable enough, i.e. no stage movement should be happening when you execute this command.
Version
3.2.1 and later
