Detailed Description

There are a few functions which control the Tescan Essence graphical user interface. Generally speaking, the remote application should not modify or use the FIB user interface, this is undesirable. On the other hand, there are situations when some interaction between the remote client and the server UI is eligible and suitable.

Get scanning status of the FIB GUI.

When combined with FibGUISetScan(), it is possible to stop the FIB scanning, execute some procedure, and then restore the scanning back to its original state.

Enable / disable the FIB GUI scanning.

If Scanning is set to 1, scanning in the topmost live window is started. If it is 0, scanning is stopped. If it is 2, a single scan is performed. See FibGUIGetScan() for details.

Parameters
Scanning0 - stop scanning, 1 - continual scanning, 2 - single scan
Timing
Executed immediately.
Call Context
Anytime
Also Affected
The result of FibGUISetScan(1) call is that continual scanning is started. The remote client must not issue any scanning requests until the scanning is stopped (either by invoking FibGUISetScan(0) or FibScStopScan()).

3.2.13 and later
