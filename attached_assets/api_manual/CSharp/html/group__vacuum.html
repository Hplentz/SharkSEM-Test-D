Get list of compatible apertures and their attributes from vacuum (also manual apertures are included)

Map of vacuum apertures is returned.

The map has the following form:

apt.0.type
apt.0.id
apt.0.diameter
apt.0.position
apt.1.type
apt.1.id
apt.1.diameter
apt.1.position
...
apt.N-1.type
apt.N-1.id
apt.N-1.diameter
apt.N-1.position
  • N is number of supported apertures.
  • type is human-readable aperture type.
  • id is a numeric identification of aperture, used in VacVPSetMode() command.
  • diameter is numeric value of aperture diameter [m]
  • position is numeric value of aperture position [m]

For future compatibility, application must silently ignore any key which does not have exactly the above form.

Returns
map containing aperture description
Timing
Executed immediately.

out int  Status

inlinestatic

Read microscope vacuum status.

The vacuum status is a single integer value. It tells the current vacuum conditions of the microscope.

It takes several minutes to evacuate the microscope. If sufficient vacuum level is not reached in an arbitrary time, the system reports error (-1).

Returns
vacuum status
-1 error - call VacGetPressure() to check the pressure
0 ready for operation
1 pumping in progress
2 venting in progress
3 vacuum off (pumps are switched off, valves are closed)
4 chamber open
Timing
Executed immediately.

◆ VacSetVPMode() [1/3]

static void SharkSEM.Commands.VacSetVPMode ( this Connection  Conn,
int  Mode,
int  ChamberGas,
int  ApertureId,
out int  Result 
)
inlinestatic

Change vacuum mode from variable pressure to high vacuum and vice versa. If automated lowvac aperture selected, be carefull that there are no devices inserted under the objective and that there is no object within 20 mm below SEM objective.

The vacuum mode change is a complex process. Some microscopes are not capable of doing the mode change automatically and user intervention is required, it may be necessary to change the aperture configuration or to open/close a valve.

Use VacGetStatus() to find out if the microscope is ready for operation after the change of the vacuum mode.

Parameters
Mode
  • 0 - high vacuum mode,
  • 1 - variable pressure mode (low vacuum)
ChamberGas- only for low vac, chamber gas
  • 0 - N2 (default)
  • 1 - H2O vapor
ApertureId- only for low vac, id of aperture
  • 0 - no aperture (default)
  • 1..N - id of aperture For S5000 and chamber ChmUniVac500, no aperture mode is not possible, there is special mode with aperture which is placed in the column.
Return values
Result
  • 0 - ok
  • -1 - invalid microscope state
  • -2 - gas is not supported
  • -3 - aperture is not supported
  • -4 - transition to low vacuum was unsuccessful - see SharkSEM Server log for more information
  • -5 - invalid parameter
  • -6 - aperture has already been in used
  • -7 - not inserted
  • -8 - decontaminator is started (postponed start)
Warning
Please make sure selected aperture is actually placed in the microscope.
Timing
Executed immediately. May take several seconds to complete the process, may even require user intervention.
Call Context
Anytime
See also
VacGetVPMode()
Also Affected
Detectors are enabled / disabled depending on the vacuum mode. HV is switched off (thermal emission) or beam is switched off (field emission) automatically.
Version
1.x.x
2.x.x
3.x.x

◆ VacVent()

static void SharkSEM.Commands.VacVent ( this Connection  Conn)
inlinestatic

Vent the microscope chamber.

Call VacVent() when chamber has to be opened. For field emission microscopes, this only allows air (nitrogen) into the chamber. On the other hand, tungsten gun microscopes have no valve between the chamber and the gun. It means that the main high voltage is automatically switched off during the venting procedure.

The call has no effect if the microscope is already vented.

Timing
Executed immediately, but venting procedure may take some time.
Call Context
Anytime
Also Affected
HV is switched off automatically.

VacGetStatus()
