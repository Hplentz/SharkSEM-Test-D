Detailed Description

DrawBeam offers rich set of functions for electron beam and ion beam patterning.

FIB systems are always equipped with DrawBeam module, other systems must have additional DrawBeam license.

For more information please refer to DrawBeam - description

inlinestatic

Load project (layer).

The XML project is self-containing, no additional information is required. It must be written in valid XML, encoded in UTF-8.

This call breaks down the project into internal binary object list ant other structures. If there is an error in XML, error code is returned.

Note that SharkSEM DrawBeam API XML project is not compatible with DrawBeam .dbp projects. SharkSEM offers only subset of DrawBeam features, on the other hand, XML projects are kept compatible (old projects are accepted by new SW versions).

Debugging of XML syntax may be difficult. Make sure that XML is valid, there are numerous XML validators available. SEM software also prints the XML parser output. In the SEM software, open Diagnostics - Health Status - Advanced - Monitoring.

Structure of the XML file is described in chapter XML Project.

inlinestatic

Pause DrawBeam process.

Only I-Etching and I-Deposition processes can be paused. When paused, beam is blanked and scanning is stopped.

I-Deposition process requires Gas Injection System (GIS) to be ready. GIS is not controlled by DrawBeam interface, it should be started using its dedicated API.

When paused, it is legal to eg. scan an image using ScScanXY(), modify some SEM or FIB parameters, etc. However, the pending exposition project cannot be modified in any way.

Returns
DrawBeam error code
Timing
Takes few tens of milliseconds till DrawBeam process is paused.
Call Context
Exposition process must be running.
Version
2.0.10 and later
3.x.x
Requires 'SharkSEM Advanced' license
