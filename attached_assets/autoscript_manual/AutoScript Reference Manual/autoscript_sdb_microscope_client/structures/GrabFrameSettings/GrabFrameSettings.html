GrabFrameSettings

Contains additional settings for a grab frame action.

Items

resolution [Optional] str The image size in pixels. You can use ScanningResolution enumeration to access standard resolution presets.
dwell_time [Optional] float The time the beam scans a single pixel.
bit_depth [Optional] int The number of bits representing one pixel. It can be either 8 or 16 bits for grayscale images or 24 for RGB images.
reduced_area [Optional] Rectangle The rectangular area to be scanned specified in the reduced area coordinate system. In this system the point [0,0] corresponds to the upper-left corner of the image and point [1, 1] to the lower-right corner.
line_integration [Optional] int If the number is > 1, each line scan is repeated several times (from 2 to 255) before proceeding to the next line. Signal data collected from these passes are integrated and shown as an actual image line. The default is 1.
scan_interlacing [Optional] int If the number is > 1, scanning splits an imaging area into blocks defined by the number of lines (from 2 to 8). In the first instance, the first line of each block is scanned, then the second one, etc. This imaging method significantly reduces sample charging. The default is 1.
preview_resolution [Optional] str The resolution of a preview image. This property is only considered for the grab_frame_to_disk() function.
frame_integration [Optional] int If the number is > 1, integration filter is switched on during image acquisition with specified number of frames.
drift_correction [Optional] bool Enables image drift correction when the integration filter is active. Requires that the frame_integration parameter is specified and is greater than 1.

Remarks

You can pass the structure as an optional parameter to functions grab_frame(), grab_multiple_frames() and grab_frame_to_disk().

For non-standard image resolutions, that is resolutions outside the predefined presets from the ScanningResolution enumeration, some of the scan settings are not applicable. The following table summarizes the situation:

  Standard resolutions Non-standard resolutions
dwell time Supported, minimum 25ns Supported, minimum 25ns
bit_depth 8, 16 and 24 8: supported on all software platform versions.
16: supported on some software platform versions.
24: not supported.
reduced_area Supported Supported
line_integration Supported, 1..255 Supported on some software platform versions, 1..255
scan_interlacing Supported, 1..8 Not supported
frame_integration Supported, 1..512 Not supported
drift_correction Supported Not supported
